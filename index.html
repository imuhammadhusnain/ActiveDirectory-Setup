Active Directory Setup on Windows Server 2019

Step-by-step guide for setting up Active Directory, DNS, and more!

1. Preparing the Windows Server 2019 Environment

Before beginning the installation, ensure you have the following:

  • A fresh installation of Windows Server 2019 (Standard or Datacenter version).
  • A static IP address (recommended for AD DS and DNS).
  • Administrative credentials for the server.

Tip: Install the latest Windows Updates to ensure the server is up-to-date.

Steps to Prepare the Server:

  • Install Windows Server 2019.
  • Configure the server's hostname and set a static IP address.

2. Installing Active Directory Domain Services (AD DS)

Active Directory Domain Services is the role that provides a central directory for user authentication and resource management.

Steps to Install AD DS:

  • Open Server Manager.
  • Click on Add Roles and Features.
  • Select Active Directory Domain Services and click Next.
  • Continue through the wizard and click Install.

3. Promoting the Server to Domain Controller

After installing AD DS, the next step is to promote the server to a Domain Controller.

Steps to Promote the Server:

  • Open Server Manager.
  • Click on the Notification Flag and click Promote this server to a domain controller.
  • Follow the wizard and choose appropriate options.

Important Note: Ensure AD CS is not set up before promoting the server to a domain controller.

Deployment Configuration:

  • Choose Add a new forest if setting up a new domain.
  • Provide a Root domain name (e.g., example.local).
  • Set the Forest functional level (recommended Windows Server 2016 or higher).

The server will now be promoted to a domain controller and reboot once the promotion is complete.

4. Installing and Configuring DNS

DNS (Domain Name System) is required for AD DS to function properly.

Steps to Install DNS:

  • Install the DNS role via Server Manager or during the promotion process.
  • Verify DNS settings in the DNS Manager.

5. Understanding Key Concepts

Active Directory

Active Directory (AD) is a directory service used to manage permissions and access to networked resources.

DNS (Domain Name System)

DNS resolves domain names to IP addresses, which is critical for AD functionality.

Domain Controller (DC)

A Domain Controller is a server that manages security authentication requests and stores the Active Directory database.

6. Group Policy (GP) & Group Policy Objects (GPO)

Group Policy (GP)

Group Policy allows administrators to manage user and computer settings across a domain.

Group Policy Objects (GPO)

GPOs are collections of settings applied to users or computers within a domain. They can be linked to organizational units (OUs), sites, or domains.

7. Top-Level Policies in Corporate Environments

  • Password Policy: Ensure strong password security.
  • Account Lockout Policy: Protect against brute-force attacks.
  • Software Restriction Policies: Control which software can run.
  • Audit Policy: Monitor and track security events.
